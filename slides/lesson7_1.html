Part 1: Vanilla RNNs

Processing sequences step by step

The Recurrent Neural Network (RNN)

At each time step \(t\), the RNN reads an input \(\mathbf{x}_t\) and updates its hidden state:

\[ \mathbf{h}_t = \tanh\!\bigl(\mathbf{W}_{xh}\,\mathbf{x}_t + \mathbf{W}_{hh}\,\mathbf{h}_{t-1} + \mathbf{b}\bigr) \]

  • \(\mathbf{h}_t \in \mathbb{R}^{d_h}\) — hidden state (the "memory").
  • \(\mathbf{W}_{xh}\) and \(\mathbf{W}_{hh}\) — shared across all time steps.
  • The same function is applied repeatedly — that's the "recurrence".

PyTorch nn.RNN

import torch
import torch.nn as nn

# input_size=64 (embedding dim), hidden_size=128
rnn = nn.RNN(input_size=64, hidden_size=128, batch_first=True)

# x: (batch, seq_len, 64)
x = torch.randn(2, 10, 64)

# output: all hidden states (2, 10, 128)
# h_n:    last hidden state  (1, 2, 128)
output, h_n = rnn(x)
  • output contains \(\mathbf{h}_1, \dots, \mathbf{h}_T\) for every time step.
  • h_n is the final hidden state — useful for classification.
  • batch_first=True makes the batch dimension come first.

Part 2: The Vanishing Gradient Problem

Why vanilla RNNs forget

Backpropagation Through Time (BPTT)

Training an RNN means unrolling all time steps and back-propagating through the chain:

\[ \frac{\partial \mathcal{L}}{\partial \mathbf{W}_{hh}} = \sum_{t=1}^{T} \frac{\partial \mathcal{L}_t}{\partial \mathbf{h}_t} \prod_{k=t}^{1} \frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}} \]

  • The product \(\prod \frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}}\) is a chain of Jacobians.
  • If each factor is < 1 → product shrinks exponentially (vanishing).
  • If each factor is > 1 → product explodes (exploding gradients).

Vanishing Gradients: Intuition

  • Imagine multiplying 0.9 by itself 50 times: \(0.9^{50} \approx 0.005\).
  • Early tokens barely influence the loss — the gradient "forgets" them.
  • The RNN can't learn long-range dependencies like: "The cat, which sat on the mat, was happy."
  • Exploding gradients can be clipped; vanishing gradients need an architecture fix.

Part 3: LSTM — Long Short-Term Memory

Gates that learn what to remember and forget

LSTM: The Key Idea

Core insight: Add a separate cell state \(\mathbf{c}_t\) that flows through time with minimal transformation — like a conveyor belt. Gates control what gets added or removed.
  • Introduced by Hochreiter & Schmidhuber (1997).
  • Three gates: forget, input, output.
  • The cell state allows gradients to flow over many time steps without vanishing.

Step 1: Forget Gate

Decides what to erase from the cell state.

\[ \mathbf{f}_t = \sigma\!\bigl(\mathbf{W}_f [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f\bigr) \]

  • \(\sigma\) outputs values in \([0, 1]\) for each dimension.
  • Values near 0 → "forget this"; near 1 → "keep this".
  • Example: when the subject changes, forget the old subject's gender/number.

Step 2: Input Gate & Candidate

Decides what new information to store in the cell state.

\[ \mathbf{i}_t = \sigma\!\bigl(\mathbf{W}_i [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i\bigr) \]

\[ \tilde{\mathbf{c}}_t = \tanh\!\bigl(\mathbf{W}_c [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_c\bigr) \]

  • \(\mathbf{i}_t\) controls how much of the new candidate to write.
  • \(\tilde{\mathbf{c}}_t\) is the proposed new content (like a vanilla RNN update).

Step 3: Update Cell State

Combine forget and input to produce the new cell state:

\[ \mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t \]

  • \(\odot\) = element-wise multiplication.
  • The forget gate erases old info; the input gate writes new info.
  • This is the "conveyor belt" — gradients can flow through \(\mathbf{c}\) without repeated nonlinearities.

Step 4: Output Gate

Decides what part of the cell state to expose as the hidden state:

\[ \mathbf{o}_t = \sigma\!\bigl(\mathbf{W}_o [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o\bigr) \]

\[ \mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t) \]

  • The cell state holds long-term memory.
  • The hidden state \(\mathbf{h}_t\) is the "working memory" exposed to the next layer / output.

LSTM Gates Summary

Gate Activation Role Analogy
Forget \(\mathbf{f}_t\) Sigmoid Erase old cell info Eraser on a whiteboard
Input \(\mathbf{i}_t\) Sigmoid Scale new candidate Deciding what to write
Candidate \(\tilde{\mathbf{c}}_t\) Tanh Proposed new content Draft notes
Output \(\mathbf{o}_t\) Sigmoid Filter cell → hidden Choosing what to say aloud

Why LSTMs Fix Vanishing Gradients

Key: The cell state update \(\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \dots\) is a linear path. Gradients flow through multiplication by \(\mathbf{f}_t\) (close to 1), not through repeated tanh squashing.
  • If the forget gate stays open (\(\mathbf{f} \approx 1\)), information persists for many steps.
  • The network learns when to remember and when to forget.
  • In practice, LSTMs can capture dependencies over 100+ time steps.

Part 4: Using LSTMs in PyTorch

From theory to code

PyTorch nn.LSTM

import torch
import torch.nn as nn

lstm = nn.LSTM(
    input_size=64,    # embedding dim
    hidden_size=128,  # hidden state size
    num_layers=1,
    batch_first=True,
)

x = torch.randn(2, 10, 64)  # (batch, seq_len, input_size)

# output: all hidden states   (2, 10, 128)
# (h_n, c_n): final hidden & cell states
output, (h_n, c_n) = lstm(x)
  • Same API as nn.RNN, but returns both \(\mathbf{h}_n\) and \(\mathbf{c}_n\).
  • num_layers stacks multiple LSTMs on top of each other.

Sentiment Classifier with LSTM

class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, emb_dim=64, hidden_dim=128):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, emb_dim)
        self.lstm = nn.LSTM(emb_dim, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, 1)

    def forward(self, ids):
        x = self.embedding(ids)           # (B, T, emb_dim)
        _, (h_n, _) = self.lstm(x)        # h_n: (1, B, hidden_dim)
        logits = self.fc(h_n.squeeze(0))  # (B, 1)
        return logits.squeeze(-1)
  • We only use the last hidden state for classification.
  • Compare with the mean-pooling classifier from Lesson 6 — LSTM preserves word order.

Stacked & Bidirectional LSTMs

Stacked (multi-layer)

  • Output of layer \(\ell\) becomes input to layer \(\ell+1\).
  • Deeper = more capacity, but slower.

Bidirectional

  • Two LSTMs: one reads left→right, one reads right→left.
  • Hidden states are concatenated: \([\overrightarrow{\mathbf{h}}_t; \overleftarrow{\mathbf{h}}_t]\).
  • Each token sees full context from both sides.
bi_lstm = nn.LSTM(
    input_size=64,
    hidden_size=128,
    num_layers=2,
    batch_first=True,
    bidirectional=True,
)

x = torch.randn(2, 10, 64)
out, (h_n, c_n) = bi_lstm(x)
# out: (2, 10, 256)  — 128 × 2 dirs
# h_n: (4, 2, 128)   — 2 layers × 2 dirs

GRU: A Simpler Alternative

The Gated Recurrent Unit (Cho et al., 2014) merges forget + input gates into a single update gate.

  • Two gates: reset \(\mathbf{r}_t\) and update \(\mathbf{z}_t\).
  • No separate cell state — just \(\mathbf{h}_t\).
  • Fewer parameters → faster to train.
  • Performance often comparable to LSTM.
# Same API, just swap LSTM → GRU
gru = nn.GRU(
    input_size=64,
    hidden_size=128,
    batch_first=True,
)

out, h_n = gru(x)  # no cell state

LSTM Limitations

  • Sequential processing: \(\mathbf{h}_t\) depends on \(\mathbf{h}_{t-1}\) — cannot parallelize across time steps.
  • Long sequences: even LSTMs struggle beyond ~500 tokens in practice.
  • Fixed bottleneck: the entire sequence is compressed into a single vector \(\mathbf{h}_T\).
  • These limitations motivated Attention and Transformers (next lesson).
